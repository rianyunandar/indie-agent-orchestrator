# packages.md - Service Delivery Packages

Use this file to package repeatable client offers for landing pages, SaaS starters, and AI web MVPs.

## Positioning Rule

Do not sell "a website". Sell a measurable business outcome:

- Landing page ready for traffic and tracking.
- AI web MVP ready for demo.
- SaaS starter ready for first users.

## Package 1: Landing Page Sprint

### Best For

- Campaign landing pages
- Service pages
- Product validation pages
- Lead generation pages
- SEO/AEO-ready content pages

### Outcome

A conversion-focused landing page that is ready to publish, track, and improve.

### Deliverables

- Landing page structure
- Conversion copy
- SEO/AEO answer block
- Title, meta description, H1, and snippet package
- CTA system
- Trust proof and objection FAQ
- Basic analytics event plan
- Launch checklist

### Skill Flow

```txt
client-delivery
-> product-discovery
-> seo-aeo
-> landing-page-conversion
-> analytics-tracking
-> launch-checklist
```

### Suggested Timeline

```txt
1-3 days for simple page
3-7 days for strategy + copy + implementation
```

### Included

- One primary page
- One primary conversion goal
- Responsive implementation
- Basic SEO/AEO setup
- Basic analytics plan
- One launch pass

### Out Of Scope

- Full brand identity
- Complex backend
- Paid ads management
- Long-form content cluster
- Advanced analytics dashboard

### Upsells

- SEO content cluster
- A/B testing plan
- Ad campaign assets
- Analytics dashboard
- Monthly conversion optimization

## Package 2: AI Web MVP

### Best For

- AI demo apps
- AI generators
- AI assistants
- Internal AI tools
- Lead magnet AI tools

### Outcome

An AI-powered web MVP with a clear use case, prompt flow, fallback behavior, and demo-ready UX.

### Deliverables

- AI feature brief
- Prompt and output schema
- UX flow
- API integration plan
- Guardrails and fallback behavior
- Basic abuse/error handling
- Analytics event plan
- Launch checklist

### Skill Flow

```txt
client-delivery
-> product-discovery
-> ai-feature-design
-> prompt-engineering
-> api-design
-> security-review
-> analytics-tracking
-> launch-checklist
```

### Suggested Timeline

```txt
3-7 days for simple AI tool
1-3 weeks for AI MVP with auth, storage, or dashboard
```

### Included

- One AI workflow
- One primary user journey
- Prompt/system instruction draft
- Structured output plan when needed
- Fallback/error states
- Basic tracking

### Out Of Scope

- Complex agent orchestration
- Long-term memory
- Enterprise compliance
- Multi-model evaluation suite
- Custom model training

### Upsells

- RAG/document search
- Prompt evaluation suite
- Admin dashboard
- Usage limits and billing
- Observability and cost dashboard

## Package 3: SaaS Starter

### Best For

- MVP SaaS products
- Internal dashboards
- Client portals
- Subscription-ready tools
- Multi-user web apps

### Outcome

A SaaS foundation ready for onboarding first users, measuring usage, and expanding into billing or operations.

### Deliverables

- SaaS scope and account model
- Dashboard UX plan
- Role/permission plan
- Onboarding flow
- API/backend structure
- Frontend implementation plan
- Analytics events
- Observability basics
- Launch checklist

### Skill Flow

```txt
client-delivery
-> product-discovery
-> saas-patterns
-> api-design
-> backend-patterns
-> frontend-patterns
-> analytics-tracking
-> observability
-> launch-checklist
```

### Suggested Timeline

```txt
1-2 weeks for lean SaaS starter
3-6 weeks for SaaS with billing, roles, and production ops
```

### Included

- Core user flow
- Dashboard shell
- Basic roles or ownership model
- Basic settings
- Analytics event plan
- Launch readiness pass

### Out Of Scope

- Advanced billing automation
- Complex team permissions
- Enterprise SSO
- Native mobile apps
- Large reporting system

### Upsells

- Payment and subscription billing
- Admin panel
- Team permissions
- Audit logs
- Error monitoring and alerting
- Backup and recovery plan

## Package Selection Guide

```md
If client needs traffic conversion -> Landing Page Sprint
If client needs an AI demo/product -> AI Web MVP
If client needs a logged-in product -> SaaS Starter
If client needs paid plans -> add Payment/Billing
If client is launching publicly -> add Launch Checklist
If client needs ongoing care -> add Maintenance
```

## Universal Client Questions

Ask these before estimating:

```md
What is the business goal?
Who is the target user?
What action should the user take?
What assets are ready?
What must be live on day one?
What is explicitly out of scope?
Who approves the work?
What is the deadline?
```

## Maintenance Offer

Use this as an optional recurring add-on:

```md
Monthly Maintenance:
- Minor content updates
- Bug fixes
- Analytics review
- SEO/AEO improvements
- Backup/health check
- Small conversion improvements
```

## Brutal Sales Rules

- Do not sell unlimited revisions.
- Do not start without a decision maker.
- Do not accept vague "make it modern" as a requirement.
- Do not build AI features without a clear input, output, and failure case.
- Do not launch a client project without analytics and a rollback path.
