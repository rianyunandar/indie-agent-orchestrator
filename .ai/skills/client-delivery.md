# Client Delivery Skill

Use this skill for freelance or agency delivery: intake, scope, revision boundaries, client communication, handover, maintenance, and acceptance.

## Core Principle

Protect the project by making expectations explicit. A good delivery system reduces confusion, revision loops, unpaid scope creep, and weak handovers.

## Load Triggers

Load this skill when the task mentions:

- client brief, proposal, scope, revision, handover, maintenance
- freelance, agency, landing page package, SaaS package, AI web project
- client approval, feedback, timeline, deliverables
- project kickoff, client onboarding, offboarding

### 3. Delivery Phases

Recommended phases:

- `Discovery`
- `Wireframe / structure`
- `Copy / content`
- `Design / implementation`
- `Review`
- `Launch`
- `Handover`

### 4. Client Feedback Control

Rules:

- Ask for consolidated feedback, not scattered messages.
- Separate bugs from change requests.
- Treat new pages, new features, and new integrations as scope changes.
- Confirm acceptance criteria before building.

Feedback format:

```md

Type: bug | revision | scope change | question
Location:

```

## Brutal Rules

- No clear deliverables, no clean project.
- No revision limit, expect endless revision loops.
- No decision maker, expect approval chaos.
- No assets deadline, project timeline is not real.
- No handover, client becomes dependent forever.
